# Repository Structure — Repo‑Map

This document describes the structure of the Repo‑Map repository.

## Root

- `README.md` — overview of the Repo‑Map repository  
- `STATUS.md` — current role and state  
- `VERSION` — repository‑level version marker  
- `CHANGELOG.md` — history of repository‑level changes  
- `CODE_OF_CONDUCT.md` — behaviour expectations  
- `CONTRIBUTING.md` — contribution process  
- `GOVERNANCE.md` — repository governance model  
- `LICENSE` — licensing and legal framing  
- `REPO-MAP.md` — canonical repository index  

## Purpose

This repository defines the **canonical repository architecture** for the SCU ecosystem.  
It is used by:

- Governance  
- Standards  
- Specifications  
- Architecture  
- Legal  
- PSCS  
- Kernel  
- All SCU Foundation programs  

It is the **source of truth** for the global refactor and all future repository alignment.
